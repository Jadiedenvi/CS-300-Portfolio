# CS-300-Portfolio

Project Overview

This repository contains work completed for CS 300: Data Structures and Algorithms. Throughout this course, I explored different data structures, analyzed their performance, and applied them to solve programming problems.

Reflection

What was the problem you were solving in the projects for this course?

The primary goal of these projects was to create a course advising program for ABC University. The program needed to read course information from a file, store the data efficiently, display an alphabetical list of courses, and allow users to search for a specific course and view its prerequisites.

How did you approach the problem? Consider why data structures are important to understand.

I approached the project by first designing solutions using pseudocode with a vector, hash table, and binary search tree. Comparing these data structures helped me understand how each one affects searching, sorting, and overall performance. After completing the analysis, I selected a binary search tree because it naturally supports ordered data while maintaining efficient searching.

How did you overcome any roadblocks you encountered while going through the activities or project?

One challenge I encountered was reading external data files correctly and ensuring the program could locate the CSV file. I also spent time debugging tree operations and verifying that courses and prerequisites were displayed correctly. Testing the program step by step helped me identify and resolve these issues.

How has your work on this project expanded your approach to designing software and developing programs?

This project reinforced the importance of planning before coding. Writing pseudocode first made the implementation much easier and helped me think about the strengths and weaknesses of different data structures before writing the final program.

How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

This course emphasized organizing code into separate functions, using meaningful variable names, and writing code that is easier to understand and modify. I also learned the value of selecting data structures based on the problem requirements rather than simply choosing the easiest implementation.
